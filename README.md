# LLM協働プロジェクト テンプレート

LLMと協働してデータドリブンな意思決定を行うプロジェクトのスターターキット。

## 使い方

1. このリポジトリをクローン
2. `PROJECT.md` を自分のプロジェクトに合わせて書き換える
3. `doc/` 内のテンプレートを必要に応じてカスタマイズ
4. フェーズ1の柱から立てていく

## 11本の柱

このテンプレートは以下の設計パターンに基づいています。
全部を最初から立てる必要はありません。フェーズに分けて段階的に導入してください。

### フェーズ1（初日に立てる）
1. **正本（PROJECT.md）** — プロジェクトの「憲法」
3. **変更管理4クラス** — インパクト別の承認プロセス
4. **検証パイプライン** — テストは「落とすため」に使う
5. **意思決定記録** — イベント+判断+結果を1レコードに
11. **撤退条件・成功指標** — やめる条件と勝ちの定義を先に決める

### フェーズ2（最初の1ヶ月で）
2. **パラメータ凍結** — いじりたい衝動をルールで封じる
6. **LLM役割分担** — オーナー/スカウト/監査役の3役
7. **プロンプト正本管理** — 固定+可変に分離してgit管理
8. **通知設計** — リスクファースト、行動を煽らない
9. **ドキュメント体系** — 読者別の複数ビュー、正本は1つ

### フェーズ3（運用しながら育てる）
10. **メモリシステム** — セッション横断の文脈維持
- 検証パイプラインの高度化
- 自動監視・自動降格

## ファイル構成

```
llm-project-template/
├── PROJECT.md                  # プロジェクト正本（穴埋め式）
├── CLAUDE.md                   # Claude Code用指示書
├── doc/
│   ├── change-classes.md       # 変更管理4クラスの詳細
│   ├── evaluation-template.md  # 外部LLM評価テンプレート
│   ├── project-context.md      # 評価LLM用コンテキスト
│   ├── retreat-conditions.md   # 撤退条件・成功指標
│   └── prompt-management.md    # プロンプト正本管理ガイド
├── logs/                       # 記録・ログ用ディレクトリ
├── .env.example                # 環境変数サンプル
├── .gitignore
└── README.md                   # このファイル
```
