# Claude Code 指示書

## 正本

このプロジェクトの正本は `PROJECT.md` です。
プロジェクトの目的・フェーズ・技術スタック・運用手順・設計判断はすべてそちらを参照してください。

このファイル（CLAUDE.md）は、Claude Codeが自動的に読み込むための派生ビューです。

---

## コーディング規約

- 日本語コメント可
- 型ヒント使用
- モジュールは単一責任で分離
- ログは logging モジュールで出力

## LLM役割分担

<!-- PROJECT.md の「LLM役割分担」セクションと整合させる -->

### Claude Code = オーナー
- コード・設計・管理の全責任
- 正本（PROJECT.md, doc/）の更新権限を持つ唯一のLLM
- 他LLMへの指示プロンプトもここで作成

### スカウト（リサーチ系LLM）
- 情報収集・定期レポートのみ
- 正本の更新はしない（提案まで）

### 監査役（評価系LLM）
- 呼び出し条件に該当する時のみ使用
- `doc/evaluation-template.md` + `doc/project-context.md` を渡す

## 通知設計の原則

- リスクファースト: 悪い情報を先に出す
- 行動の選択肢に「何もしない」を含める
- 数字と判断を分離: データは客観的に、判断は人間に委ねる
- 記録と行動を分離: ボタンは「記録」であって「実行」ではない

---

## データガバナンス

- 外部LLMに渡してよいデータの範囲を明示する
- 機密データ（個人情報・財務データ等）は外部LLMに直接渡さない
- データの加工・匿名化ルールを定める
- 監査ログを残す（誰が何をいつ承認したか）

---

## 重要な約束事

- **変更管理:** 変更は4クラスに分類し、クラスに応じた手順で進める
- **凍結ルール:** チェックポイント到達まで対象パラメータは変更しない
- **テストは「落とすため」に使う:** 同じデータでの無限最適化はしない
- **記録を残す:** 見送りも含めて全ての判断を記録する
- **撤退条件は先に決める:** 感情ではなく数字で判断する
- **ドキュメントの整合性:** 重要な変更コミット前に全docをgrepして整合性チェック
