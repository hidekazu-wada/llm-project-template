# 初期化ヒアリング手順（Claude Code 用）

このファイルは Claude Code が**テンプレートを新規プロジェクト用に最適化する際に従う手順書**です。
ユーザーから「このテンプレを最適化して」「このプロジェクトを初期化して」などの依頼を受けたら、ここに書かれた流れでヒアリングを進めてください。

---

## 進め方の原則

- **対話的に進める。** 一度に投げる質問は **3〜5問まで**。回答を受けて次の質問に進む
- **憶測で埋めない。** 答えが曖昧なら聞き直す
- **ヒアリング中は実装しない。** PROJECT.md の書き換えは、全質問が終わってから一括で提案する
- **質問の順序は守る** — 後半の質問は前半の答えに依存する

---

## ヒアリング項目（全8項目）

### ラウンド1: プロジェクトの根幹（3問）

1. **プロジェクトの目的** — どんな問題を解決したいか。1〜2文で
2. **プロジェクト名と対象ユーザー** — 名前と、個人利用か組織利用か
3. **想定する成果物の形** — 例: BOT、CLI、Web アプリ、自動化スクリプト、リサーチレポート、ドキュメント生成器、など

### ラウンド2: 技術と体制（3問）

4. **関わるLLM** — オーナー（Claude Code）以外に、どのLLMをどの役割（スカウト／監査役）で使うか
5. **技術スタックの希望** — 言語、実行環境（ローカル／クラウド／cron）、データ保存先（ファイル／DB／クラウドストレージ）、通知先（Slack／Discord／メール／なし）
6. **機密データの扱い** — 外部LLMに渡してよい情報の範囲、渡してはいけない情報

### ラウンド3: 成功と撤退（2問）

7. **成功指標の初期案** — 何をもって「このプロジェクトは成功した」と言えるか。測定可能な形で
8. **撤退条件の初期案** — 何が起きたら / どの数字を割ったらやめるか

---

## ヒアリング完了後のアクション

全質問の回答が揃ったら、ユーザーに以下を**まとめて提案**し、承認を得てから実行する:

1. **`PROJECT.md` の各セクションを埋める提案**
   - ヒアリング結果から自動で埋められる項目を埋めた完成形を提示
   - 不確実な箇所は `<!-- TBD -->` で残す

2. **不要な doc/ ファイルの削除提案**
   - 例: 監査役LLMを使わないなら `doc/evaluation-template.md` と `doc/project-context.md` は不要
   - 例: 外部LLMを使わないなら同上

3. **`.env.example` の書き換え**
   - 使うLLM・通知先に合わせて環境変数を整理

4. **`CLAUDE.md` 冒頭の「初期化モード」セクションの削除**
   - 初期化が完了したら、このトリガー文言は不要なので削除する
   - 同時にこのファイル（`doc/init-questionnaire.md`）も削除

5. **README.md の更新**
   - 「テンプレートの使い方」ではなく、初期化済みプロジェクト向けの説明に書き換え

6. **初回コミット**
   - 「プロジェクト初期化」というメッセージで、上記の変更をコミット

---

## 注意

- ヒアリング中に判明した重要な決定事項は、PROJECT.md の「重要な設計判断の履歴」にも初回エントリとして追加する
- ユーザーが「あとで決める」と言った項目は、PROJECT.md にプレースホルダを残しておく（強引に埋めない）
